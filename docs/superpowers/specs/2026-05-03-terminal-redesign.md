## Overview

Redesign pisanvs.cl as a terminal-themed personal site. The UI looks and feels like a Unix shell вЂ” the user "explores files" вЂ” but the actual commands are buttons on the screen. The page loads with all content visible; no blank prompt. A real prompt bar at the bottom exists for power users who want to type.

All copy lives in a `content/` folder of Markdown files. A build script generates `content.js` from them. GitHub Actions runs the script on every push to `main` before GitHub Pages deploys.

## Terminal UI

Faithfully implements `ui_kits/terminal/index.html` from the design bundle.

**Visual tokens (from design system):**
- Background: `#000`
- Foreground: `#d8d8d8`
- Green: `#0f0` / dim: `#6c6` / faint: `rgba(0,255,0,.25)`
- Cyan: `#18BCBC`
- Orange: `#E58D3C`
- Muted: `#888`
- Font: Roboto Mono (variable, self-hosted from `assets/fonts/`)

**Chrome elements:**
- `PISANVS BIOS v2.0` вЂ” fixed top-left, orange, 11px
- `в—Џ online В· last commit Xh ago` вЂ” fixed top-right, green, blinking dot
- CRT scanline overlay вЂ” `repeating-linear-gradient`, pointer-events none, z-index 10
- CRT canvas вЂ” animated grid lines, opacity 0.12

**Index page (pre-rendered, always visible):**
1. Header: `# pisanvs` (orange) + one-liner bio + `[read the manual]` button
2. ASCII portrait (right column, green-dim, 7px monospace)
3. `## projects` вЂ” numotics, tedx, blog each with `[cat README]` chip + `[./run в†—]` chip
4. `## tools` вЂ” chordid, mic-al with `[./run в†—]` chips
5. `## code` вЂ” commit graph (53Г—7 deterministic pseudo-random) + link to `~/code/`
6. `## elsewhere` вЂ” inline links to github, instagram, linkedin, substack, mail + gpg button
7. `## now` вЂ” live status line + `[tail -f]` button

**Prompt bar (bottom, fixed):**
```
pisanvs@cl:~/code$ _
```
Supports: `ls`, `cd`, `cat`, `pwd`, `whoami`, `socials`, `history`, `clear`, `./exec`, `help`. Arrow keys for history, Ctrl+L to clear.

**Interaction model:**
- All content visible on load вЂ” no blank terminal
- Buttons append output below in `#log`, never clearing above
- `cd` auto-runs `ls` and shows clickable directory listing
- `./run` execs open an inline pane with close button

**Animations:**
- Fade-in on load for each index block (staggered, 0.6s)
- Blinking dot on status indicator
- No other motion вЂ” consistent with brand

## Content Pipeline

### `content/` folder
Plain Markdown files. No frontmatter required for most. Authors edit these directly.

### `scripts/build-content.js`
- Node.js, zero external dependencies (uses `fs`, `path` from stdlib)
- Reads every `.md` / `.asc` file under `content/`
- Outputs `content.js` with a `const CONTENT = { ... }` object matching the shape used by `fs.js`
- File names map to keys: `content/projects/numotics.md` в†’ `CONTENT.projects.numotics`
- Run locally: `node scripts/build-content.js`

### GitHub Actions (`build.yml`)
```
on: push (branch: main)
jobs:
  build:
    - checkout
    - setup-node (v20)
    - run: node scripts/build-content.js
    - commit & push content.js if changed
  (GitHub Pages deploys automatically from main)
```
